Instructions

We want to evaluate PCMs extracted from informal product descriptions (overviews). To this end, MatrixMiner provides ways to visualize the PCM with a traceability with the original product descriptions (at the left) and the technical specification (at the right). We display one column at a time and you have to attribute scores for the feature and cell values.
Please follow these instructions:

Feature:
  • look at each feature in the PCM and state whether the feature is : well formed, not meaningful, irrelevant or redundant. You can also edit the feature.
  • specify whether for this feature, the overlap of the overview or the specification is bigger, or the features are incomparable.
    Note: for each cell value, the corresponding product overview is depicted with the highlight of the feature name and value in the text.
Cell values:
  • look at each cell value in the PCM and state whether the expected cell value is correct, incorrect, incomplete or missing. You can also edit the cell value.
  • specify for each cell value if the overlap of the overview or the specification is bigger; or the cell values are incomparable.
    Note: for each cell value, the corresponding specification is depicted.
Column:
  • look at each column in the PCM and provide an overall rating for the column as a whole.
    A score of 5 means that the column is cohesive, while a score of 1 means that the group is not meaningful at all.
    Incoherent or not cohesive columns means that all cell values belong together in this column.
